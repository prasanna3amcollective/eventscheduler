# Plan: Own Responsibility Form (Similar to Create Activity)

## Goal
Implement a "Own Responsibility" flow that mirrors the existing "Create Activity" experience:
- A button in the calendar toolbar ("Own Responsibility")
- Clicking it opens a modal with a form
- Submitting the form creates a new `Responsibility` record via API
- The UI should feel consistent with the existing Activity creation flow

## Current State
- Button exists in `CalendarView.tsx:102` with placeholder `onClick`
- No modal, form, or API endpoint exists yet for Responsibilities
- `Responsibility` model in Prisma schema is very similar to `Activity` (name, start/end datetime, duration, isRecurring, recurrenceRule, category, state)

## Proposed Changes

### 1. Backend - New API Route
- Create `src/app/api/responsibilities/route.ts`
- Implement `POST` handler that:
  - Accepts payload similar to Activity (name, startDateTime, endDateTime, duration, isRecurring, recurrenceRule, category)
  - Uses `withAuth` + `getSessionContext` for authorization
  - Creates a `Responsibility` record
  - Returns 201 on success

### 2. Frontend - New Form Component
- Create `src/components/ResponsibilityForm.tsx`
- Reuse or adapt logic from `ActivityForm.tsx` (date pickers, recurring options, validation)
- On submit, call the new `/api/responsibilities` endpoint
- Accept props: `initialData`, `onResponsibilityCreated`, `onCancel`

### 3. Modal Wrapper
- Reuse the existing `ActivityModal` component (it is generic)
- Or create a thin `ResponsibilityModal` if needed for future differentiation

### 4. Integration in Main Page (`src/app/page.tsx`)
- Add state:
  - `isResponsibilityModalOpen`
  - `selectedResponsibility` (for future editing)
- Add handler `onOwnResponsibility()` that sets default values and opens the modal
- Render the new modal + form inside the page (similar to how `ActivityModal` + `ActivityForm` are rendered)
- Pass `onOwnResponsibility` down to `CalendarView`

### 5. Calendar Toolbar (`src/components/CalendarView.tsx`)
- Add new prop `onOwnResponsibility?: () => void` to `CalendarViewProps`
- Update `CustomToolbar` to receive and use `onOwnResponsibility`
- Wire the existing "Own Responsibility" button to call `onOwnResponsibility`

### 6. Styling & UX
- Use the same color scheme and layout patterns as `ActivityForm`
- Button color already set to `#d97757` (distinct from primary)
- Form should support the same fields as Responsibility model (including recurring)

## Open Questions / Tradeoffs
- Should `ResponsibilityForm` be a separate component or reuse/extend `ActivityForm`?
- Do we need edit functionality for Responsibilities immediately, or just create?
- Should Responsibilities appear on the calendar, or only in a separate list?

## Files to Modify/Create
- `src/app/api/responsibilities/route.ts` (new)
- `src/components/ResponsibilityForm.tsx` (new)
- `src/app/page.tsx` (add handler + modal)
- `src/components/CalendarView.tsx` (add prop + wire button)

## Implementation Order
1. Create API route first (so form can call it)
2. Create ResponsibilityForm
3. Wire everything in page.tsx and CalendarView
4. Test create flow

This plan keeps the implementation close to the existing Activity pattern for consistency.
