# Migration Plan: Replace Global CSS with Tailwind CSS (Zero Visual Change)

## Goal
Migrate from the 2600+ line `src/app/globals.css` (with hundreds of custom classes) to Tailwind CSS, while ensuring **identical visual appearance** — no pixel-level differences in UI.

## Current State
- Next.js 16 + React 19 app
- `src/app/globals.css` (2644 lines) defines:
  - CSS custom properties (design tokens: colors, fonts, shadows, radii)
  - Global resets
  - Typography rules
  - Hundreds of component-specific classes (`.btn-primary`, `.modal-overlay`, `.admin-table`, `.form-group`, etc.)
  - Complex selectors, hover/focus states, responsive rules, animations
- All components use `className="..."` with these custom classes (370+ usages across ~20 TSX files)
- No Tailwind currently installed
- Fonts loaded via Google Fonts + Geist in layout

## High-Level Strategy
1. Install + configure Tailwind for Next.js (v4 recommended for Next 16)
2. Extract all design tokens from `:root` into Tailwind `theme.extend`
3. Convert global styles and class definitions to Tailwind utilities + a minimal `globals.css` (only for unavoidable base styles, @import, and complex animations)
4. Incrementally replace every custom class usage in components with equivalent Tailwind classes
5. Use visual regression / side-by-side comparison to guarantee zero visual change
6. Remove unused custom CSS after migration

## Detailed Steps

### Phase 1: Setup (Non-breaking)
- Run `npx tailwindcss init -p` (or use Next.js Tailwind CLI if available)
- Update `tailwind.config.ts`:
  - Content paths: `./src/**/*.{js,ts,jsx,tsx,mdx}`
  - Extend `theme` with all existing CSS vars:
    - colors (primary, surface, text, error, etc.)
    - fontFamily (heading, body, mono)
    - borderRadius, boxShadow, etc.
  - Add any custom plugins if needed for glassmorphism/backdrop-filter
- Add Tailwind directives to `globals.css` (keep existing imports + tokens temporarily)
- Update `layout.tsx` to include Tailwind classes on html/body if needed
- Run `npm run dev` and verify app still renders exactly as before

### Phase 2: Token Extraction & Base Styles
- Move font imports and CSS var definitions into Tailwind config or keep a thin layer in globals.css
- Keep only these in final globals.css:
  - Google Fonts import
  - Root CSS vars (or convert to Tailwind theme)
  - Body/html base styles
  - Any non-replicable global rules (e.g., specific ::selection, scrollbar)
- Convert all typography rules (h1-h6, time, .mono) into reusable Tailwind classes or @apply in a small layer

### Phase 3: Component-by-Component Migration (Iterative)
For each component file, identify custom classes and replace:
- Common patterns:
  - `.btn-primary` → Tailwind equivalent (bg-primary text-white px-... rounded-... shadow-... hover:...)
  - `.modal-overlay` / `.modal-content` → fixed inset-0 bg-black/50 flex items-center justify-center + inner card styles
  - Form elements: `.form-group`, `.form-row`, `.input-error` → flex, gap, border-red-500 etc.
  - Tables, admin panels, pagination → grid/table utilities + custom variants
  - Badges, cards, sections → bg-surface border rounded shadow etc.
- Use `@apply` in globals.css for very long repeated utility groups (to keep JSX clean)
- Preserve all inline styles and dynamic className logic (e.g. conditional error classes)
- For animations (`fade-in`, transitions) either keep minimal CSS or replicate with Tailwind transitions

Order of migration (recommended):
1. Simple components (LoginForm, RegisterForm, Icons)
2. Modals (ActivityDetailModal, ResponsibilityDetailModal, ProfileModal)
3. Forms (ActivityForm, ResponsibilityForm)
4. Views (CalendarView, ActivityCarousel)
5. Complex admin (AdminDashboard — many tables/forms)
6. Main page + layout wrappers

### Phase 4: Validation & Cleanup
- After each batch of changes, run the app and visually diff (or use Percy/Playwright screenshots)
- Use browser devtools to compare computed styles before/after for critical elements
- Once all custom classes are removed from JSX, delete corresponding rules from globals.css
- Run `npm run build` and `npm run lint` to catch issues
- Remove any dead CSS (can use `purge` verification or Tailwind's built-in tree-shaking)
- Final globals.css should be < 200 lines

## Risks & Mitigations
- Risk: Complex nested selectors or pseudo-elements hard to replicate
  - Mitigation: Keep minimal CSS for those cases; use Tailwind `arbitrary variants` or `before:/after:`
- Risk: Glassmorphism / backdrop-filter + custom shadows
  - Mitigation: Map exactly to Tailwind arbitrary values or extend theme
- Risk: Font loading / sizing differences
  - Mitigation: Explicitly set font sizes/weights using the same values from original CSS

## Success Criteria
- `globals.css` reduced from 2644 → minimal base only
- Zero visual difference when comparing production builds side-by-side
- All components use Tailwind classes (or @apply wrappers)
- App passes lint + typecheck + builds cleanly

## Estimated Effort
- 4–8 hours for experienced Tailwind + Next.js dev (mostly mechanical replacement + verification)

This plan ensures a clean, maintainable Tailwind setup with **no regression** in appearance.
