# Plan: Category Filter Strip for Upcoming Activities

## Goal
Add a horizontal strip of clickable category pills between `carousel-header` and `carousel-strip-wrapper` in `ActivityCarousel.tsx`. Clicking a pill filters the displayed activities to that category. Only show pills for categories that have activities within the next 12 months.

---

## Files to modify

### 1. `src/components/ActivityCarousel.tsx`

**Add import:**
- `addMonths` from `date-fns` (already a dependency in `package.json`)

**Add state:**
```ts
const [filterCategory, setFilterCategory] = useState<string | null>(null);
```

**Derive filtered activity list:**
- Create a `useMemo` for `activitiesNext12Months` — filter `upcomingActivities` to those whose `startDateTime` is within 12 months from now.
- Create a `useMemo` for `categoryList` — unique, sorted categories from `activitiesNext12Months`.
- Create `filteredActivities` — if `filterCategory` is null, use `upcomingActivities`; otherwise filter by `filterCategory`.

**Render the category strip** between `</div>` (closing `carousel-header`) and `<div className=...carousel-strip-wrapper...>`:
- Wrap in `<div className="carousel-category-strip">`
- Only render if `categoryList.length > 1` (single category or none = no need to filter)
- Map `categoryList` into pill buttons:
  - `<button className={pill-classes} onClick={() => setFilterCategory(category === filterCategory ? null : category)}>`
  - Active pill gets class `active` for highlighted style

**Use `filteredActivities`** instead of `upcomingActivities` when rendering the cards inside `carousel-strip`.

**Activity count** in header should reflect filtered count: `<span>{filteredActivities.length} activities</span>`. Or keep it showing the total? The user didn't specify, but it makes sense to show filtered count for consistency. Keep total count in header, show filtered count elsewhere? Actually, simplest: update count to show `filteredActivities.length` since that reflects what's visible.

## Edge cases

- **Loading state**: No category strip shown (component returns early).
- **0 activities**: No category strip shown (only the "No highlights" empty state).
- **1 category**: Strip hidden (no filtering needed).
- **Many categories**: Strip scrolls horizontally (overflow-x: auto).
- **Active category + no matching activities**: The existing empty state "No highlights scheduled..." will show.
- **Category pills only from next 12 months**: Uses `activitiesNext12Months` to derive categories. Activities beyond 12 months still appear in the carousel when no filter is active.
