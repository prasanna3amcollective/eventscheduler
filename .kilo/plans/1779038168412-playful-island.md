   - Use `setConfirmMessage(...)` and `setConfirmAction(() => async () => { ... actual delete logic ... })`

   - On save error (line 464): show inline error banner or reuse existing error state

4. Render the modal JSX at the end of the component (copy/adapt from AdminDashboard:1248-1275), adjusting styles if needed for consistency.

## Task 2: Fix Phone Validation in RegisterForm

### Context
- `RegisterForm.tsx:37`: `const PHONE_REGEX = /^\d{10}$/;`
- Used at line 136 for validation
- Error message at line 33 hard-codes "10 digits"
- Country selector exists (lines 39-50+) but validation ignores selected dial code

### Recommended Approach (choose one)
**Option A (Preferred - Minimum length check)**: Replace rigid regex with a simple length check (e.g. `phone.length >= 7 && phone.length <= 15`) after stripping non-digits. This supports international numbers without complex per-country logic.

**Option B (Country-aware)**: Store selected country and use different regex/length rules per dial code. More accurate but higher maintenance.

### Changes Required
1. Update `ERROR_MESSAGES.INVALID_PHONE` to reflect new rule (e.g. "7-15 digits").
2. Modify validation logic around line 136:
   - Strip non-digit characters from phone input.
   - Apply chosen rule (minimum length or country-based).
3. Update the phone input handling if needed to allow international formats.
