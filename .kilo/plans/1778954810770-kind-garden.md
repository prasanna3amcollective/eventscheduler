# Plan: Owner as User Typeahead + Index User.name

## Goal
- Change the `owner` field in ResponsibilityForm from plain text input to a UserTypeahead dropdown (suggests users after 3+ characters typed).
- Add database index on `User.name` for performance on typeahead searches.
- Keep `owner` stored as string (user name) for backward compatibility.

## Changes Overview
- Prisma schema: Add index on User.name.
- Run migration.
- Update UserTypeahead.tsx to only suggest/filter when query.length >= 3 (configurable threshold).
- Refactor ResponsibilityForm.tsx:
  - Import UserTypeahead + User type.
  - Fetch users list via /api/users (like ActivityForm).
  - Replace owner input with `<UserTypeahead ... />` using formData.owner.
  - Handle single string value (name) via onChange.
- Minor: Ensure ResponsibilityData interface supports it (already does as string).
- No API or backend changes needed (POST already accepts owner: string|null).
- Update default in page.tsx if needed (already has owner: '').

## Detailed Steps

### 1. Schema Update (Index)
- In `prisma/schema.prisma`, update User model:
  ```prisma
  name     String @index
  ```
- Or use `@@index([name])` for composite if future needed.
- Run migration: `npx prisma migrate dev --name index-user-name`

### 2. UserTypeahead Enhancement
- Modify `src/components/UserTypeahead.tsx`:
  - Add constant `MIN_CHARS = 3`.
  - In handleInputChange and onFocus: only setFilteredUsers / setIsOpen if val.length >= MIN_CHARS.
  - Update placeholder hint if desired (e.g. "Type at least 3 characters...").
  - This makes suggestions appear only after three characters, improving UX/performance.

### 3. ResponsibilityForm Refactor
- Add imports: `useEffect`, `UserTypeahead`, `User as UserIcon`.
- Add User interface (copy from ActivityForm or import type).
- Add state: `const [users, setUsers] = useState<User[]>([]);`
- useEffect to fetch users once: same as ActivityForm lines ~297-309.
- In JSX form-grid, replace the owner `<div className="form-field..."><input ...` with:
  ```tsx
  <UserTypeahead
    label="Owner"
    value={formData.owner}
    onChange={(val) => setFormData(prev => ({ ...prev, owner: val }))}
    icon={<UserIcon size={16} />}
    users={users}
    placeholder="Type 3+ chars to search users..."
  />
  ```
- Keep formData shape unchanged (owner remains string).
- Update handleSubmit to still send `owner: formData.owner.trim() || null`.

### 4. Integration / Polish
- `src/app/page.tsx`: No change needed (default already includes owner: '').
- Ensure styling matches (typeahead uses .form-group but form uses .form-field; may need minor CSS tweak or wrap, but ActivityForm uses it directly in grid so compatible).
- Test: Open Own Responsibility modal, type 3 chars in Owner → see dropdown suggestions, select → saves name.

### 5. Tradeoffs / Notes
- Owner remains denormalized string (name) not FK to User.id (keeps simple, matches original plan).
- Index improves lookup speed for typeahead without full-text search.
- Typeahead allows free-text fallback (user can type non-matching name if needed).
- No overlap checks added for responsibilities (v1 scope).

## Bug Fix: "Unknown argument `category`"
The error occurs because the Prisma Client (`@prisma/client`) is out of sync with the current schema (stale generated types do not recognize the `category` field on Responsibility even though it exists in schema.prisma).

**Root cause**: After schema changes/migrations, `prisma generate` was not (re)run, so the extended Prisma client in `src/lib/prisma.ts` uses outdated types.

**Resolution**:
- After any `prisma migrate dev` or `db push`, always run:
  ```
  npx prisma generate
  ```
- Add to implementation order: Run `prisma generate` immediately after migration.
- Optionally add a postinstall script or package.json script for it.

This will resolve the create responsibility failure.
